# Changelog: v1.5.0 — 2024-03-15

---

## User-Facing Changes

### ⚠️ Breaking Changes

**API: Removed `color` field from user response**
The `color` field has been removed from `GET /api/users` and `GET /api/users/{id}`
responses. This field was deprecated in v1.4.0.

**Migration**: Replace `user.color` with `user.theme.primary_color` in your integration.
```json
// Before (v1.4 and earlier)
{ "id": "usr_abc", "color": "#3B82F6", ... }

// After (v1.5+)
{ "id": "usr_abc", "theme": { "primary_color": "#3B82F6" }, ... }
```

---

### ✨ New Features

**Dark mode**
The app now supports dark mode. Follows your system preference automatically,
or set your preference in Settings → Appearance. (#251)

**CSV export for reports**
Export any report as a CSV file from the report view. Use the "Export" button
in the top-right corner of any report. Large reports are processed in the
background — you'll receive an email with a download link when ready. (#234)

---

### 🚀 Improvements

**Reports generate 40% faster**
Report generation time has been significantly reduced through database query
optimization. A report that previously took 10 seconds now completes in ~6 seconds.
No action needed. (#267)

---

### 🐛 Bug Fixes

**Dashboard loading spinner no longer hangs on empty data**
The loading spinner on the dashboard now correctly stops when a dataset returns
no results. Previously it would spin indefinitely. (#241)

**Users with special characters in name can update their profile**
Users whose names contain accented characters (é, ñ, ü, etc.) were unable to
save profile updates due to an encoding issue. This is now fixed. (#261)

---

### 🕐 Deprecations

**API v1 Reports endpoint deprecated**
`GET /api/v1/reports` is deprecated and will be removed in **v2.0** (Q2 2024).

**Migrate to**: `GET /api/v2/reports`
See [API v2 migration guide](https://docs.example.com/api/v2-migration) for full details. (#256)

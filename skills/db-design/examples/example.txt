# Example: /db-design

## User Input
Design the database schema for a SaaS project management tool. Users belong to
organizations. Organizations have projects. Projects have tasks. Tasks can be
assigned to users, have comments, attachments, and labels. Need to support
searching tasks by label, assignee, and status. Multi-tenant — orgs must be
fully isolated.

## Skill Output

# Database Design: SaaS Project Management

## Domain Model
- Organization 1:N Users (membership via org_members)
- Organization 1:N Projects
- Project 1:N Tasks
- Task N:M Users (via task_assignees)
- Task 1:N Comments
- Task 1:N Attachments
- Task N:M Labels (via task_labels)
- Label belongs to Project (labels are project-scoped)

## Schema

```sql
-- Multi-tenancy root
CREATE TABLE organizations (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name        VARCHAR(255) NOT NULL,
    slug        VARCHAR(100) NOT NULL,
    plan        VARCHAR(20) NOT NULL DEFAULT 'free'
                    CHECK (plan IN ('free', 'pro', 'enterprise')),
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    CONSTRAINT uq_org_slug UNIQUE (slug)
);

-- Users (global, cross-org via memberships)
CREATE TABLE users (
    id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email        VARCHAR(255) NOT NULL,
    display_name VARCHAR(100) NOT NULL,
    avatar_url   TEXT,
    created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    CONSTRAINT uq_users_email UNIQUE (email)
);

-- Org membership (role scoped to org)
CREATE TABLE org_members (
    org_id     UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
    user_id    UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    role       VARCHAR(20) NOT NULL DEFAULT 'member'
                   CHECK (role IN ('owner', 'admin', 'member', 'viewer')),
    joined_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (org_id, user_id)
);

CREATE TABLE projects (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    org_id      UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
    name        VARCHAR(255) NOT NULL,
    description TEXT,
    status      VARCHAR(20) NOT NULL DEFAULT 'active'
                    CHECK (status IN ('active', 'archived')),
    created_by  UUID NOT NULL REFERENCES users(id),
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    CONSTRAINT uq_project_name_per_org UNIQUE (org_id, name)
);

CREATE TABLE tasks (
    id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id   UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    org_id       UUID NOT NULL REFERENCES organizations(id),  -- denormalized for query perf
    title        VARCHAR(500) NOT NULL,
    description  TEXT,
    status       VARCHAR(20) NOT NULL DEFAULT 'todo'
                     CHECK (status IN ('todo', 'in_progress', 'in_review', 'done', 'cancelled')),
    priority     VARCHAR(10) NOT NULL DEFAULT 'medium'
                     CHECK (priority IN ('low', 'medium', 'high', 'urgent')),
    due_date     DATE,
    created_by   UUID NOT NULL REFERENCES users(id),
    created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Task assignees (N:M)
CREATE TABLE task_assignees (
    task_id    UUID NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
    user_id    UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    assigned_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (task_id, user_id)
);

-- Labels are project-scoped
CREATE TABLE labels (
    id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    name       VARCHAR(50) NOT NULL,
    color      CHAR(7) NOT NULL DEFAULT '#6B7280',  -- hex color
    CONSTRAINT uq_label_name_per_project UNIQUE (project_id, name)
);

CREATE TABLE task_labels (
    task_id    UUID NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
    label_id   UUID NOT NULL REFERENCES labels(id) ON DELETE CASCADE,
    PRIMARY KEY (task_id, label_id)
);

CREATE TABLE comments (
    id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    task_id    UUID NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
    author_id  UUID NOT NULL REFERENCES users(id),
    body       TEXT NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE attachments (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    task_id     UUID NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
    uploaded_by UUID NOT NULL REFERENCES users(id),
    filename    VARCHAR(255) NOT NULL,
    storage_key TEXT NOT NULL,          -- S3 key
    size_bytes  INTEGER NOT NULL,
    mime_type   VARCHAR(100) NOT NULL,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

## Indexes

```sql
-- Tenant isolation + task list queries
CREATE INDEX idx_tasks_project_id       ON tasks(project_id);
CREATE INDEX idx_tasks_org_id_status    ON tasks(org_id, status);
CREATE INDEX idx_tasks_due_date         ON tasks(due_date) WHERE due_date IS NOT NULL;

-- Assignee filtering (very common query)
CREATE INDEX idx_task_assignees_user_id ON task_assignees(user_id);

-- Label filtering
CREATE INDEX idx_task_labels_label_id   ON task_labels(label_id);

-- Comments and attachments access
CREATE INDEX idx_comments_task_id       ON comments(task_id);
CREATE INDEX idx_attachments_task_id    ON attachments(task_id);

-- Org member lookup
CREATE INDEX idx_org_members_user_id    ON org_members(user_id);
```

## Key Query Patterns

```sql
-- Tasks for a project, filtered by assignee and label
SELECT DISTINCT t.* FROM tasks t
JOIN task_assignees ta ON ta.task_id = t.id
JOIN task_labels tl ON tl.task_id = t.id
WHERE t.project_id = $1
  AND ta.user_id = $2
  AND tl.label_id = $3
  AND t.status != 'cancelled'
ORDER BY t.created_at DESC
LIMIT 20;

-- All tasks assigned to a user across all their orgs
SELECT t.* FROM tasks t
JOIN task_assignees ta ON ta.task_id = t.id
JOIN org_members om ON om.org_id = t.org_id AND om.user_id = ta.user_id
WHERE ta.user_id = $1 AND t.status = 'in_progress';
```

## Design Decisions

**`org_id` denormalized onto `tasks`**: Avoids JOIN through projects every time we
need tenant isolation. Small write overhead, large read benefit.

**Labels are project-scoped, not org-scoped**: Teams in the same org may use
different label systems per project. Org-wide labels can be added in v2 if needed.

**No soft deletes**: CASCADE DELETE used throughout. Audit log handled separately.

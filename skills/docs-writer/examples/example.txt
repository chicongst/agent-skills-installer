## Skill Output

# File Upload API

Upload files to be stored and served via CDN. Supports images and PDFs up to 10 MB.

## Quick Start

```bash
curl -X POST https://api.example.com/v1/files \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@photo.jpg"
```

**Response:**
```json
{
  "data": {
    "id": "file_abc123",
    "url": "https://cdn.example.com/files/file_abc123.jpg",
    "filename": "photo.jpg",
    "size": 204800,
    "mime_type": "image/jpeg",
    "created_at": "2024-01-15T10:30:00Z"
  }
}
```

## Prerequisites

- Valid API token with `files:write` scope
- File must be one of: `image/png`, `image/jpeg`, `application/pdf`
- File size must not exceed **10 MB**

## Endpoint

**`POST /v1/files`**

Upload a single file using multipart form data.

### Request

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `file` | File (multipart) | Yes | The file to upload |

**Headers:**
| Header | Value |
|--------|-------|
| `Authorization` | `Bearer {token}` |
| `Content-Type` | `multipart/form-data` (set automatically by HTTP clients) |

### Response

**201 Created** — Upload successful

```json
{
  "data": {
    "id": "file_abc123",
    "url": "https://cdn.example.com/files/file_abc123.jpg",
    "filename": "original-name.jpg",
    "size": 204800,
    "mime_type": "image/jpeg",
    "created_at": "2024-01-15T10:30:00Z"
  }
}
```

### Errors

| Status | Code | Cause |
|--------|------|-------|
| 400 | `FILE_TOO_LARGE` | File exceeds 10 MB limit |
| 400 | `UNSUPPORTED_TYPE` | File type not in allowed list |
| 400 | `NO_FILE` | No file included in request |
| 401 | `UNAUTHORIZED` | Missing or invalid token |
| 413 | `PAYLOAD_TOO_LARGE` | Request body exceeds server limit |

**Error response format:**
```json
{
  "error": {
    "code": "FILE_TOO_LARGE",
    "message": "File size 15.2 MB exceeds the 10 MB limit"
  }
}
```

## Code Examples

**JavaScript (fetch)**
```javascript
async function uploadFile(file, token) {
  const formData = new FormData();
  formData.append('file', file);

  const response = await fetch('https://api.example.com/v1/files', {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${token}` },
    body: formData,
  });

  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.error.message);
  }

  return response.json();
}
```

**Python (requests)**
```python
import requests

def upload_file(file_path: str, token: str) -> dict:
    with open(file_path, 'rb') as f:
        response = requests.post(
            'https://api.example.com/v1/files',
            headers={'Authorization': f'Bearer {token}'},
            files={'file': f}
        )
    response.raise_for_status()
    return response.json()['data']
```

## Troubleshooting

**`FILE_TOO_LARGE` even for small files**
Check that you're not accidentally sending the file as base64 inside JSON — use
multipart form data, not JSON body. Base64 encoding increases file size by ~33%.

**`UNSUPPORTED_TYPE` for a valid file**
The type is detected from file content (magic bytes), not the filename extension.
A `.jpg` file saved as PNG internally will be rejected. Re-export the file correctly.

**Upload succeeds but URL returns 403**
The CDN URL is publicly accessible — if you're getting 403, the upload may still be
processing. Wait 2-3 seconds and retry. For immediate access, use the `url` field
only after a successful 201 response.
